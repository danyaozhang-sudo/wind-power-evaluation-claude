### 1.2 利用小时数获取（硬性规定）

> ⚠️ **先区分两个概念**：
> - **理论可利用小时数**(h) = 基于测风数据/风资源图谱推算的项目预期利用小时数。通常由年平均风速、风功率密度和所选风机机型功率曲线共同决定。可参考项目所在地附近风电场（同地形/同机型）的数据。**这是输入财务模型的值，但还不包含限电扣减**。
> - **实际可利用小时数**(h) = 近2年省内同区域/同类型风电场的实际平均运行小时数，**已包含限电、弃风和停机检修的综合影响**。用于校核限电率。
> 
> 隐含限电率 = `1 - (实际可利用小时数 / 理论可利用小时数)`。如果这个值明显高于搜索到的实际限电率，说明限电可能比预想严重。

利用小时数分**两步**获取：

**第一步：取低值输入财务模型**
- 通过 `web_search` 全方位搜索（不少于10个不同类型的来源），查询分析得到**理论可利用小时数**
- 从 Obsidian 知识库提取**理论可利用小时数**
- 两者取低值 → 直接输入财务模型（保守原则）

**第二步：实际利用小时数校核**
- 通过 `web_search` 全方位搜索（不少于10个不同类型的来源），查询分析得到项目所在地近两年的**实际可利用小时数**（与第一步的搜索不是同一口径）
- 从 Obsidian 知识库提取**实际可利用小时数**
- 两者取低

**第三步：交叉验证限电率**
- 用理论值和实际值计算隐含限电率 = `1 - (实际小时数 / 理论小时数)`
- 与通过限电率估算（见第三步）得到的限电率交叉验证
- 若两者接近 → 数据可信；若差异大 → 需分析原因，取保守值

**第四步：最终输入**
- 理论可利用小时数（取低值）输入财务模型
- 实际利用小时数用于校核限电率

---

#### ⚠️ 关键：机制电量的基准不是项目实际小时数，而是省定基准小时

**这是最常见的错误假设**。各省确定机制电量的逻辑可能完全独立于项目的实际可利用小时数：

1. **多数省采用省定基准利用小时数**（而非项目实际测风小时数）：
   - 省发改委/能源局每年发布该省风电/光伏的**基准利用小时数**（如新疆风电895h、安徽风电待查）
   - 机制电量 = 装机容量 × **省定基准小时** × 机制电量比例
   - **省定基准小时 ≠ 项目可利用小时数**，两者可以相差很大
   - 例：新疆2025年风电优先发电计划小时仅**895h**（保量保价部分），而项目实际可利用小时可达2,000-3,000h
   - 此时即使机制比例80%，实际受益电量也仅 = 装机×895h×80%，远超低于装满发小时数

2. **机制电量确定方式各省差异极大**，至少有三种计算口径：

  | 计算口径 | 代表省 | 说明 |
  |:--------|:-------|:-----|
  | **省定基准小时法** | 存量项目（多省通用） | 机制电量 = 装机 × 省定基准小时 × 比例 |
  | **上网电量百分比法** | 新疆增量（50%）、广东（80%）、山东（80%） | 机制电量 = 年实际发电量 × 比例。**136号文增量项目多数省采用此方式** |
  | **绝对电量分配法** | 宁夏、天津 | 全省总分配额度内按项目分配 |

  **正确做法（三步走）**：
  - 第一步：`{省份} 136号文 机制电价 实施细则` → 找到官方文件
  - 第二步：文件中明确写"按其上网电量的X%" → 百分比法；写"按省定基准利用小时" → 基准小时法
  - 第三步：根据确认的口径计算：百分比法→发电量×比例；基准小时法→min(基准小时,实际小时)×比例

3. **搜索方法**：
   - `{省份} 风电 优先发电 计划 利用小时`（查找省定基准小时）
   - `{省份} 风电 保障性 收购 小时数`
   - `{省份} 风电 年度 优先发电 电量`
   - 对于省定基准小时明确的省份，**优先用省定小时×机制比例** 计算机制电量收入

4. **财务模型处理**：
   - 如果省定基准小时 < 项目利用小时 → 机制电量 = 省定基准小时 × 机制比例
   - 差额电量（总发电量 - 省定基准小时×机制比例）进入市场化交易
   - 如果省定基准小时不可得，**保守假设**无省定基准优惠，全额走市场化+机制比例

## 第三步：确定关键参数

### 限电率估算（双轨并行）

**轨道A——项目所在地评估限电率**：
结合接入方案、断面阻塞、消纳能力和省内区域位置：
- **基础三档**：
  - ① 消纳能力≥年发电量1.2倍、无断面阻塞 → 2%-3%
  - ② 消纳能力为1-1.2倍、轻微断面阻塞 → 4%-6%
  - ③ 消纳能力<1倍、严重断面阻塞 → 15%-20%
- **省内区域调整系数**：
  - 同省内，项目位于**消纳较好区域**（近负荷/近外送汇集站/省内南部/东部）→ 在基础档位下限取值
  - 项目位于**消纳较差区域**（远负荷/近电源集中区/省内西部/北部）→ 在基础档位上限取值
  - 调整幅度一般在基础值的 ±15-30%
- **示例**：
  - 甘肃酒泉：全省消纳③（15-20%），酒泉属河西电源集中区取上限 → 18-20%
  - 甘肃陇南：全省消纳③，陇南属河东较好区取下限，且可降一档至② → 10-15%

**轨道B——全省同类型电源平均限电率**（新增）：

### 电价策略

**情况一**（不参与机制电价竞价）：
- 年度中长期交易 70%，按当地中长协均价
- 现货交易 30%，按当地现货均价
- 绿电交易 100%覆盖（与中长协/现货叠加），按绿电均价
- 扣除两个细则考核（0.015元/kWh）和市场费用分摊（结算电费×2%）

**情况二**（参与机制电价竞价）：
- ⚠️ **关键假设确认（必须做）**：在评估前，先确认以下两个政策参数，切勿默认100%：

  **① 机制电量比例**（`mechanism_ratio`）：
  - 不是100%电量都享受机制电价！136号文规定增量项目"单个项目申请纳入机制的电量，可低于其全部发电量"
  - 各省比例差异极大：山东/广东80%、甘肃上限80%、辽宁55%、宁夏仅10%
  - **搜索**：`{省份} 机制电价 机制电量 比例 上限` 或查看 Obsidian `topics/全国各省机制电价竞价结果汇总.md`
  - 未查到具体比例时，参考同类省份值并取保守（较低比例）
  - `mlt_ratio` = 机制电量比例，不可设1.0

  **①-1 🚨 机制电量基数的致命陷阱——先查计算口径！**  
  确定机制电量时，**基数可能不是项目实际小时数，也不是省定基准小时——必须查文件确认计算口径**。这是最致命的错误，一旦弄错收入偏差可达300%+。各省136号文实施细则可能有三种口径：

  | 场景 | 机制电量计算方式 | 典型省份 |
  |:----|:---------------|:---------|
  | 省定基准小时法 | 机制电量 = 装机×省定基准小时×机制比例 | 新疆（895h）、河南、辽宁 |
  | 总发电量百分比法 | 机制电量 = 年实际发电量×机制比例 | 山东、广东、甘肃 |
  | 绝对电量分配法 | 全省总分配额度内按项目分配 | 宁夏、天津 |

  **正确做法（三步走）**：
  - 第一步：搜索该省136号文实施细则：`{省份} 136号文 机制电价 实施细则`
  - 第二步：从文件中确认机制电量计算口径（百分比法 vs 基准小时法 vs 分配法）
  - 第三步：根据确认口径代入模型
  - 如未能找到官方文件 → 暂用百分比法（上网电量×比例），并在报告中标注"未找到136号文细则，按上网电量百分比法计算"

  **② 机制电价执行期限**（`mechanism_years`）：
  - 136号规定增量项目执行期限"按回收初始投资的平均期限确定"
  - 各省通行标准：**10-12年**（四川12年、甘肃12年、宁夏10年、辽宁12年）
  - **搜索**：`{省份} 机制电价 执行期限`
  - 执行期满后，全部电量进入市场化交易（中长期+现货），不再享受差价保护

- **差额电量处理**：未纳入机制的电量（比例 = `1 - mechanism_ratio`）只能参与市场化交易
  - 大概率走**现货市场**，按当地风电现货均价结算
  - 执行期满后（第 `mechanism_years+1` 年起），全部电量按市场化电价结算

- **电费计算（二阶段法）**：
  ```
  机制期内（前 N 年）：
    加权均价 = mechanism_ratio × 机制电价 + (1 - mechanism_ratio) × 现货价
  市场化期（后 20-N 年）：
    加权均价 = MKT_long_ratio × 中长期价 + MKT_spot_ratio × 现货价
  ```
  - 均扣除两细则（0.015元/kWh）和市场费（2%）
  - N = 机制电价执行期限（mechanism_years）

### 增值税抵扣规则（更新为融资租赁模式）

风电项目采用融资租赁模式，增值税进项税抵扣规则：

每年增值税销项税以**上网电价全额**为税基（不含罚款、市场费扣减）：
- 销项税 = 年发电量 × 上网电价(含税) × 13% / 1.13
- 进项税可抵扣 = 当年（**本金+利息**）× 13% + 运维费 × 13%（运维备件为主适用13%税率）
- 可抵扣金额可结转下一年使用，直至融资偿还完毕

**全投资FCF特殊处理**：全投资自由现金流计算时使用**无杠杆附加税**（`add_tax_unlevered`），即剔除利息进项税的影响，以确保全投资口径不受融资结构干扰。详见 `scripts/financial_model.py` 中的 `add_tax_unlevered` 数组。

### ⚠️ 数据缺失时的估计方法（2026-05-11新增）

**优先级一：邻省参考法**

- 中长期电价参考2026年全国中长协下行趋势（江苏0.344、浙江0.345、广东0.372）

### ⚠️ 机制电价参数陷阱（重要）

**陷阱1：mlt_ratio=1.0** — 切勿将 `mlt_ratio` 默认为1.0（100%电量按机制电价）。实际机制电量比例通常为**47%-80%**，差额部分走现货。设1.0会高估收入15-30%。

**陷阱2：全生命周期统一电价** — 机制电价有执行期限（10-12年），之后全部进入市场化。若用20年统一电价，会严重高估后期收入。

**🚨 陷阱3（最致命）：搞错机制电量计算口径**
这是最常见且影响最大的错误。各省136号文实施细则对机制电量的计算口径不统一，混用会导致收入偏差300%+。

| 错误假设 | 实际情况 | 后果 |
|:---------|:---------|:-----|
| 默认百分比法，新疆项目2,800h×50%=1,400h受机制保护 | 查文件发现新疆用的是百分比法→正确；但如果山东用基准小时法(895h)误用百分比法(2,800h×80%) | 高估或低估 |
| 山东项目→用项目实际小时算机制电量 | 先查山东136号文细则，确定是百分比法还是基准小时法 | 口径不对全错 |
| 河南项目→直接用百分比法 | 河南可能有省定基准小时法 | 需要查文件确认 |

**判断规则**：
1. 必须找到该省**136号文实施细则**，确认机制电量的计算口径（是百分比法还是基准小时法）
2. **省定基准小时法**：如果政策写的是按"省定基准利用小时"核定 → 机制电量 = min(省定基准小时, 项目利用小时) × 机制比例。搜索：`{省份} 风电 优先发电 计划 利用小时`
3. **上网电量百分比法**：如果政策写的是"按其上网电量的X%" → 机制电量 = 年实际发电量 × 比例。这是136号文多数省份采用的方式
4. 两种方式的收入结果差异极大：基准小时法下机制电量远低于百分比法（如新疆优惠小时895h vs 实际2,800h）
5. 如果未找到该省136号文细则 → 暂用实际小时×机制比例（百分比法默认），注明"省定基准小时未查到，按上网电量百分比法计算"

**陷阱4：混淆"申报比例"和"中标比例"**
- 申报时填的比例（80%）≠ 实际获得的机制电量比例（47%）
- 边际出清机组获得的机制电量比例远低于申报值
- 以实际中标公示数据为准，而非申报文件中的数据

**陷阱5（新增）：混淆"扣费前现货价"和"扣费后现货价"**
- 搜索到的现货均价通常为**扣费前**的结算价（如新疆Q1 0.1723元/kWh）
- 两细则(-0.015)和市场费(-2%)应在加权均价上**一次性扣除**，不应：①仅对现货部分扣市场费 ②对现货扣后再整体扣两细则（导致双重扣减或逻辑不一致）
- 正确做法：加权均价 → 统一扣两细则 → 统一扣市场费。财务模型 `financial_model.py` 的二阶段逻辑已内置此计算
- 错误案例（2026-05-11塔城）：先对现货价×0.98=0.1689，再加权平均后-0.015 → 得出0.1850，与模型计算的0.1715不一致 → 模型更准确

**陷阱6（新增）：现货电价季节偏差**
- Q1新疆风电出力大、负荷少 → 现货价全年最低
- Q3空调用电高峰 → 现货价全年最高
- Q1数据不可用于全年评估，必须取年度均值或做敏感性范围

**解决方案——financial_model.py 三阶段电价改造**：

`financial_model.py` 默认用单一 `effective_price` 计算所有年份。对参与机制电价的增量项目，需改造为**三阶段电价**：

```python
# ── 三阶段电价参数 ──
use_three_stage_pricing = True

# 阶段一：机制期（前 N 年）
# 这部分电量分为机制电量（按机制价）和差额电量（走市场）
mechanism_ratio = 0.47       # 机制电量比例（实际中标比例，非申报比例）
mechanism_price = 0.21       # 机制电价 元/kWh
mechanism_years = 10         # 执行期限

# 省定基准小时（用于核定机制电量的上限）
# 注意：是省定基准小时，不是项目实际利用小时！
province_base_hours = 895    # 如新疆风电省定基准895h。填0表示未查到
project_actual_hours = 2800  # 项目实际可利用小时数

# 阶段二：机制期满后（后 20-N 年），全部市场化
mkt_long_ratio = 0.60
mkt_long_price = 0.215
mkt_spot_ratio = 0.40
mkt_spot_price = 0.19

# ── 三阶段收入计算逻辑（在 calculate() 中）──
for i in range(operating_years):
    if i < mechanism_years:
        # 阶段一：机制期
        if province_base_hours > 0:
            # 省定基准小时法：机制电量=基准小时×机制比例
            mech_hours = min(province_base_hours, project_actual_hours)
            mech_kwh = capacity_kw * mech_hours * mechanism_ratio
```
